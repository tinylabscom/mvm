Install

One-liner (latest release)
curl -fsSL https://raw.githubusercontent.com/auser/mvm/main/scripts/mvm-install.sh | sh
Pin a version
MVM_VERSION=v0.3.0 curl -fsSL https://raw.githubusercontent.com/auser/mvm/main/scripts/mvm-install.sh | sh
From source
cargo install mvm
Self-update
mvm upgrade

Quick Start

# Set up the dev environment (Lima VM with Nix + Firecracker):
mvm dev

# Build a microVM image from a Nix flake:
mvm build --flake .

# Boot a headless Firecracker VM:
mvm run --flake . --cpus 2 --memory 1024

# Or use templates for reusable images:
mvm template create my-app --flake .
mvm template build my-app

# Check VM health and security posture:
mvm vm ping
mvm security status

How It Works

Three-Layer Stack

The CLI runs on your host, Lima provides a Linux VM with /dev/kvm, and Firecracker runs microVMs inside it.

Nix-Based Builds

Reproducible microVM images from Nix flakes. Builds are cached — rebuilds are near-instant when nothing changed.

Headless MicroVMs

No SSH, ever. MicroVMs communicate via Firecracker vsock. The guest agent handles health, status, and lifecycle.

Integration Health

Any workload registers health checks via drop-in JSON files. The guest agent polls them and reports via mvm vm status.

Templates & Registry

Build reusable base images, version them, and share via an S3-compatible registry with template push/pull.

Security Posture

Evaluate jailer isolation, seccomp filters, network isolation, and audit logging with mvm security status.

Integration Health Checks

Any workload running inside a microVM can register health checks with the mvm guest agent. Drop a JSON file in /etc/mvm/integrations.d/ and the agent monitors it automatically.

// /etc/mvm/integrations.d/my-service.json
{
  "name": "my-service",
  "health_cmd": "systemctl is-active my-service",
  "health_interval_secs": 15,
  "health_timeout_secs": 5
}

For NixOS images, use the services.mvm-integrations module to declare integrations declaratively. The guest agent is auto-injected into every mvm-built image.

Prerequisites

macOS or Linux host with Homebrew installed. Running mvm bootstrap handles everything else — Lima, Nix, and Firecracker are installed automatically inside the VM.

Or install Lima manually and use mvm setup to configure the rest.
